# Инструкция по настройке GPT-OSS-120B

## Обзор

Интеграция локальной модели GPT-OSS-120B позволяет анализировать данные и генерировать графики через AI.

## Требования

1. Модель GPT-OSS-120B должна быть скачана на компьютер
2. Python 3.8+
3. Установленные зависимости из `requirements.txt`

## Установка зависимостей

```bash
pip install -r requirements.txt
```

## Настройка модели

### Вариант 1: Использование llama-cpp-python (Рекомендуется для GGUF моделей)

1. Установите `llama-cpp-python`:
```bash
pip install llama-cpp-python
```

2. Убедитесь, что модель в формате GGUF (например, `gpt-oss-120b.gguf`)

3. Создайте файл `.env` в корне проекта и добавьте настройки:
```env
# Путь к модели (абсолютный или относительный путь)
GPT_MODEL_PATH=C:/path/to/your/gpt-oss-120b.gguf

# Тип модели
GPT_MODEL_TYPE=llama-cpp

# Размер контекста (по умолчанию 4096)
GPT_N_CTX=4096

# Количество потоков для CPU (по умолчанию 4)
GPT_N_THREADS=4

# Количество слоев для GPU (0 = только CPU)
GPT_N_GPU_LAYERS=0
```

### Вариант 2: Использование transformers (для PyTorch моделей)

1. Установите необходимые зависимости:
```bash
pip install transformers torch
```

2. В `.env` файле укажите:
```env
GPT_MODEL_PATH=C:/path/to/your/model
GPT_MODEL_TYPE=transformers
```

### Вариант 3: Использование vLLM (для API сервера)

1. Запустите vLLM сервер отдельно:
```bash
python -m vllm.entrypoints.openai.api_server --model /path/to/model
```

2. В `.env` файле укажите:
```env
GPT_MODEL_TYPE=vllm
VLLM_API_BASE=http://localhost:8000/v1
```

## Настройка для Windows

### Пример .env файла для Windows:

```env
# Модель GPT-OSS-120B
GPT_MODEL_PATH=C:\Users\vmare\models\gpt-oss-120b.gguf

# Тип модели (llama-cpp, transformers, vllm)
GPT_MODEL_TYPE=llama-cpp

# Параметры модели
GPT_N_CTX=4096
GPT_N_THREADS=4
GPT_N_GPU_LAYERS=0
```

**Важно:** Используйте прямой путь (backslashes) или двойные обратные слеши в Windows:
- `C:\Users\vmare\models\model.gguf` ✅
- `C:/Users/vmare/models/model.gguf` ✅
- `C:\\Users\\vmare\\models\\model.gguf` ✅

## Проверка работы

1. Запустите Django сервер:
```bash
python manage.py runserver
```

2. Откройте в браузере: `http://localhost:8000/ai-analysis/`

3. На странице AI анализа проверьте статус модели - должно быть "Готово к работе"

## Использование

### Анализ данных

1. Перейдите на страницу "AI Анализ" в меню
2. (Опционально) Введите вопрос для анализа
3. Нажмите "Запустить анализ"
4. Дождитесь результата анализа

### Генерация графиков

1. На странице "AI Анализ"
2. Нажмите "Сгенерировать график"
3. AI предложит оптимальный тип графика на основе данных
4. График будет отображен автоматически

## Устранение проблем

### Ошибка: "Модель не настроена"

- Проверьте, что `GPT_MODEL_PATH` указан в `.env` файле
- Убедитесь, что путь к модели правильный и файл существует

### Ошибка: "Не удалось инициализировать модель"

- Проверьте формат модели (должен быть GGUF для llama-cpp)
- Убедитесь, что установлены все зависимости
- Проверьте логи Django для подробной информации об ошибке

### Модель загружается медленно

- Для больших моделей (120B) загрузка может занять несколько минут
- Убедитесь, что у вас достаточно RAM
- Для GPU ускорения установите `GPT_N_GPU_LAYERS` больше 0

### Ошибка импорта библиотек

```bash
# Для llama-cpp-python
pip install llama-cpp-python

# Для transformers
pip install transformers torch

# Для vLLM
pip install vllm
```

## Производительность

- **llama-cpp-python**: Лучше для CPU, поддерживает GGUF формат
- **transformers**: Требует больше памяти, но может использовать GPU
- **vLLM**: Оптимально для больших моделей с GPU, требует отдельный сервер

## Примечания

- Модель загружается при первом использовании (lazy loading)
- Для больших моделей может потребоваться значительный объем RAM
- Рекомендуется использовать GPU для ускорения работы с моделью 120B
